promise详解

promise generate async 的关系

  1. promise 基础
  2. generater 基于 Promise 加了一个执行流程
  3. async 是 generater 的语法糖
    • async 函数 他的返回值是一个 promise 对象
      • 如果需要获取 async 函数的返回值，那么有两种方式：
        • then 接收
        • 在另一个 async 函数中使用 await 来接收
    • await 等待一个异步操作的结果，后面跟一个promise对象才有意义
    • 在async 函数中用 try{}catch(err){} 来捕获异常

promise

  1. 一个 promise 有三种状态：
    • 正在做
    • 已完成
    • 已失败
  2. 是一个容器，用来封装一个异步操作,而他自己本身也是一个异步任务
  3. 体现在程序中就是一个构造函数，这个构造函数接收一个参数
    • 该参数为函数，作为promise 容器来装异步操作的代码
    • 该函数有两个参数 resolve reject ，异步操作成功调用 resolve() 方法，失败调用reject()方法
    • 该构造函数 new 以后会得到一个promise 对象，这个对象可以进行 .then() 操作
      • .then()函数有两个参数，都是函数，
      • 第一个函数在调用了resolve()方法后会执行，若没有调用resolve(),则第一个函数不再执行，这个参数函数的形参为 resolve() 函数的实参
      • 第二个函数在调用了 reject() 函数以后会执行
      • 可以连续 .then() 下一个 .then()的函数的参数是上一个 .then() 的函数 return 的值(针对return的是普通数据而言)
      • .then()的参数函数 return 的数据若为promise对象，那么下一个 .then() 的 参数函数将作为return 出去的promise对象的 resolve 处理函数

修改第三方库中组件的默认样式

vue项目中修改第三方库中组件的默认样式

​ 最近做的vue项目中使用的是第三方组件库是 iview ，今天在使用它里面的折叠面板时想要修改一下样式，弄了半天勉勉强强算是达到目标了，但是方法不是很好。下午boss开会碰巧讲到了这个问题，并且给了解决方法，不得不感叹，boss就是boss。

​ 废话不多说，直接开始。

样式私有化

  • 在vue组件中，为了使样式私有化（模块化），不对全局造成污染，可以在 style 标签上添加 scoped 属性以表示它只属于当下的模块。但是我们在使用第三方组件库去修改它的默认样式时，会有很多不便。

    ​

    弄懂原理就知道为什么修改默认样式这么难了！

    scoped私有化样式的原理

  • 给HTML的DOM节点加一个不重复data属性(形如：data-v-2311c06a)来表示他的唯一性

  • 在每句css选择器的末尾（编译后的生成的css语句）加一个当前组件的data属性选择器（如[data-v-2311c06a]）来私有化样式

  • 如果组件内部包含有其他组件，只会给其他组件的最外层标签加上当前组件的data属性

    因此，第三方组件库的样式权重都特别高，要想修改就得设置更高的权重

    解决办法

​ 下面讲第一种，也就是boss的解决办法，第二种是我自己的。

  1. 添加 id 法

    • 为目标的元素或他的父级元素添加id属性，在设置样式时使用 id 选择器，因为 id 具有唯一性，所以不用担心该样式会造成全局污染。

      目标元素可以为HTML元素，也可以为第三方库中的元素

    注意在 style 标签中不需要添加 scoped 属性

    ​

  2. js修改style法

    使用 js 语法来修改元素的样式优先级要高于内联式的样式

    • 直接获取元素，使用 js 语句去操作 样式
      • 给对应的元素，或者其他元素设置 ref 属性，通过 this.$refs.值 来获取元素

        注意：若 ref 加在 第三方库的 组件身上，获取到的元素为被包装过的元素，并不是原生的元素，所以最好给原生的HTML元素设置 ref 属性，可以通过访问关系的方法定位到目标元素
