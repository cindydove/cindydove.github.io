上午总结

  • 回调函数
    • 把一个函数作为参数传递给函数调用，在函数内部调用参数函数，这个被传递的函数就叫做回调函数。
    • 同步回调和异步回调
    • 同步回调没有意义，多此一举
    • 只有当需要获取一个函数中异步操作的结果的时候，我们才必须使用回调函数来获取
  • 处理回调函数中的异常
    • 结论：对于异步操作中的异常，不要在内部处理，应该调用 callback 往外抛出
    • 由调用者来决定如何处理这个异常
    • 一般在 Node 中的异步操作都会伴随异常的发生，所以我们遵循一个约定：
    • ：错误优先，把错误对象作为回调函数的第一个参数往外抛出
    • ：由调用者通过判断第一个参数是否是 null 来决定如何处理异常
  • 将这里学到的知识运用到 guestbook 案例中了
