vue中的 watch

watch的基本用法

  • watch属性的作用：监视数据 的变化，从而做一件事情；VUE实例身上有watch属性；
    • 定义：一个对象，键是需要观察的表达式，值是对应回调函数。
    • 什么时候使用deep深度监视呢：1. 监视的数据是一个对象； 2. 而且这个对象并没有被重新赋值，只是修改了对象的属性
    • 使用 watch 还可以监视表达式，如果监视的是表达式，则前面的 表达式，必须使用引号包裹起来；
    • ajax获取的数据会有延迟性(异步请求的原因)，造成数据还没获取到位但是页面已经加载完毕，此时要在页面中进行展示的数据为undefined,可以用watch监视Ajax获取的数据的变化进行重新渲染页面
    • watch 中回调函数不能用箭头函数，只能用普通的函数形式
      var vm = new Vue({
          el: '#app',
          data: {
              a: 1,
              user: {
                  name: 'zs',
                  age: 18
              },
          },
          methods: {},
          router,
          watch: {
            // a也可以用引号包裹
              a: function (newValue，oldValue) { 
              },
              'user.name': function (newValue，oldValue) {              
              },
              user: {
                handler: function (newval) { // 如果监视到了 user 的变化，则，调用 handler 处理函数
                console.log(newval)
                },
                deep: true, // deep 表示深度监视   【用的不多】
                immediate: true //实现页面刚进入的时候，强制调用一下`watch`
              }
              // watch 除了可以监视 单个变量名之外；还可以监视 表达式，表达式必须用引号包裹
              '$route.path': function (newVal, oldVal) {                           
              }
          }
      });

父组件将异步操作返回的值传递给子组件并使用 watch 监听

  1. 父组件将异步操作中返回的值传递给子组件，而子组件使用该值时会发现值为 undefined
  2. 原因：由于该值为异步操作获取的，所以在组件进行渲染时该值可能并没有被返回，此时传入的值就是 undefined
  3. 解决办法：利用 watch 对数据进场监听,当数据发生变化时，再进行操作
